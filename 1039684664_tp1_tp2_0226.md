# vLLM TP1 Baseline 與 TP2 A/B 測試報告（0226）

## 1) 測試目標

- 先取得可用基線：TP=1、ShareGPT、2048/128、concurrency=512
- 再做 TP=2 版本 A/B 穩定性對照

## 2) TP1 Baseline（有效）

測試檔：`/root/vllm-bench/bench_sharegpt_2048_128_c512_r24_tp1_baseline.json`

條件：

- vLLM 0.16.0（host venv）
- `tensor-parallel-size=1`
- `dataset=sharegpt`, `input-len=2048`, `sharegpt-output-len=128`, `ignore-eos`
- `num-prompts=512`, `request-rate=24`, `max-concurrency=512`

結果：

- completed: 512
- failed: 0
- request_throughput: 7.5612 req/s
- output_throughput: 967.8343 tok/s
- total_token_throughput: 2772.7471 tok/s
- mean_ttft_ms: 22099.15
- mean_tpot_ms: 31.65
- total_output_tokens: 65536

換算：

- RPM = 7.5612 x 60 = 453.67

## 3) TP2 A/B 結果（穩定性）

### A 組：vLLM 0.16.0（host）

證據：`/root/vllm-bench/vllm_server_clean2.log`

觀察：

- 反覆出現 `TimeoutError: RPC call to sample_tokens timed out`
- 後續出現 `EngineDeadError`
- benchmark 會出現 `output tokens = 0` 或服務中途失效

### B 組：vLLM 0.9.0.1（docker）

容器：`vllm/vllm-openai:v0.9.0.1`（TP=2, GPU0+1）

觀察：

- 啟動階段即失敗，`NCCL error: unhandled system error`
- `Engine core initialization failed`
- 無法進入可壓測狀態

## 4) 結論

1. `decode throughput = 0` 在本案不是正常值，屬於 TP=2 路徑異常造成的無效結果。
2. 目前在此主機環境下，TP=2 於兩個版本皆不穩：
   - 0.16.0：可啟動但推理期間 engine timeout/dead
   - 0.9.0.1：NCCL 初始化直接失敗
3. 可用且可重現的有效數據目前是 TP=1 baseline。
