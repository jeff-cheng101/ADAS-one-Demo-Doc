# vLLM 多卡部署重測報告（0302_2）

## 1. 測試說明

本次依使用者要求，使用既有來源結果檔的測試條件重新進行多卡部署測試。

- 指定來源（使用者提供）：`/root/vllm-bench/bench_sharegpt_2048_128_c512_r24.json`
- 實際可用來源（同條件檔案）：`/root/vllm-bench/test1/bench_sharegpt_2048_128_c512_r24.json`

> 原因：指定路徑檔案不存在，因此採用同名同條件的 `test1` 來源檔作為重跑基準。

## 2. 測試條件

- 模型：`mistralai/Mistral-7B-Instruct-v0.3`（FP16）
- 多卡設定：`tensor-parallel-size=2`（GPU0+GPU1）
- 資料集：ShareGPT
- 請求條件：輸入 2048 tokens、輸出 128 tokens
- 壓測條件：`request-rate=24`、`max-concurrency=512`、`num-prompts=512`

## 3. 執行命令

### 3.1 服務啟動

```bash
CUDA_VISIBLE_DEVICES=0,1 vllm serve mistralai/Mistral-7B-Instruct-v0.3 \
  --dtype half \
  --tensor-parallel-size 2 \
  --max-model-len 2304 \
  --gpu-memory-utilization 0.75 \
  --max-num-seqs 4 \
  --host 127.0.0.1 \
  --port 8000
```

### 3.2 Benchmark

```bash
vllm bench serve \
  --backend openai-chat \
  --base-url http://127.0.0.1:8000 \
  --endpoint /v1/chat/completions \
  --model mistralai/Mistral-7B-Instruct-v0.3 \
  --dataset-name sharegpt \
  --dataset-path /root/vllm-bench/ShareGPT_V3_unfiltered_cleaned_split.json \
  --input-len 2048 \
  --sharegpt-output-len 128 \
  --ignore-eos \
  --num-prompts 512 \
  --request-rate 24 \
  --max-concurrency 512 \
  --disable-tqdm \
  --save-result \
  --result-dir /root/vllm-bench \
  --result-filename bench_sharegpt_2048_128_c512_r24_from_source_20260302_082500.json
```
