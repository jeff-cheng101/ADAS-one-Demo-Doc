# vLLM 清場後雙卡重跑報告（0226 TP2 Clean Rerun）

## 測試目標

在下列情境驗證是否達標：

- 輸入 2,048 tokens / 輸出 128 tokens
- Concurrency = 512
- 門檻
  - 解碼吞吐量 >= 3,000 tokens/s
  - 單請求平均速度 >= 10 tokens/s
  - 系統處理能力 >= 1,400 RPM

## 清場流程

1. 停止所有 vLLM 相關行程（包含殘留 `vllm.entrypoints.openai.api_server`）。
2. 確認 `nvidia-smi`：GPU0/GPU1 只剩 Xorg，顯存回到 ~11 MiB。
3. 重新啟動雙卡 TP=2 服務後再跑 benchmark。

## 測試配置

- 模型：`mistralai/Mistral-7B-Instruct-v0.3`（FP16）
- 服務啟動：

```bash
CUDA_VISIBLE_DEVICES=0,1 vllm serve mistralai/Mistral-7B-Instruct-v0.3 \
  --dtype half \
  --tensor-parallel-size 2 \
  --max-model-len 4096 \
  --gpu-memory-utilization 0.9 \
  --max-num-seqs 64 \
  --host 127.0.0.1 --port 8000
```

- 基準命令：

```bash
vllm bench serve \
  --backend openai-chat \
  --base-url http://127.0.0.1:8000 \
  --endpoint /v1/chat/completions \
  --model mistralai/Mistral-7B-Instruct-v0.3 \
  --dataset-name sharegpt \
  --dataset-path /root/vllm-bench/ShareGPT_V3_unfiltered_cleaned_split.json \
  --input-len 2048 \
  --sharegpt-output-len 128 \
  --ignore-eos \
  --num-prompts 512 \
  --request-rate 24 \
  --max-concurrency 512 \
  --disable-tqdm \
  --save-result \
  --result-dir /root/vllm-bench \
  --result-filename bench_sharegpt_2048_128_c512_r24_tp2_clean.json
```
