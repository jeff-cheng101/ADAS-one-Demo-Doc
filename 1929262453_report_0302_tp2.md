# vLLM 多卡部署測試報告（0302）

## 1. 測試目的

驗證 vLLM 在以下條件下之推論效能，並對照目標門檻：

- 測試情境：每請求輸入 2,048 tokens、輸出 128 tokens
- 併發數：512
- 目標門檻：
  - 解碼吞吐量 >= 3,000 tokens/s
  - 單請求平均速度 >= 10 tokens/s
  - 系統處理能力 >= 1,400 RPM

## 2. 測試環境與條件

- 模型：`mistralai/Mistral-7B-Instruct-v0.3`（FP16）
- 服務：vLLM（TP=2，多卡）
- 資料集：ShareGPT（`/root/vllm-bench/ShareGPT_V3_unfiltered_cleaned_split.json`）
- Benchmark：`vllm bench serve`

## 3. 執行命令

### 3.1 多卡服務啟動

```bash
CUDA_VISIBLE_DEVICES=0,1 vllm serve mistralai/Mistral-7B-Instruct-v0.3 \
  --dtype half \
  --tensor-parallel-size 2 \
  --max-model-len 2304 \
  --gpu-memory-utilization 0.75 \
  --max-num-seqs 4 \
  --host 127.0.0.1 \
  --port 8000
```

### 3.2 壓測命令

```bash
vllm bench serve \
  --backend openai-chat \
  --base-url http://127.0.0.1:8000 \
  --endpoint /v1/chat/completions \
  --model mistralai/Mistral-7B-Instruct-v0.3 \
  --dataset-name sharegpt \
  --dataset-path /root/vllm-bench/ShareGPT_V3_unfiltered_cleaned_split.json \
  --input-len 2048 \
  --sharegpt-output-len 128 \
  --ignore-eos \
  --num-prompts 512 \
  --request-rate 24 \
  --max-concurrency 512 \
  --disable-tqdm \
  --save-result \
  --result-dir /root/vllm-bench \
  --result-filename bench_sharegpt_2048_128_c512_r24_tp2_rerun_20260302_073952.json
```

## 4. 測試結果

來源檔案：`/root/vllm-bench/bench_sharegpt_2048_128_c512_r24_tp2_rerun_20260302_073952.json`

- Completed: 512
- Failed: 0
- Request throughput: 2.4152 req/s
- Output throughput（解碼吞吐量）: 309.1497 tok/s
- Total token throughput: 885.6826 tok/s
- Mean TTFT: 95003.57 ms
- Mean TPOT: 12.59 ms
- Total output tokens: 65536
- Duration: 211.99 s

衍生指標：

- RPM = `2.4152 * 60 = 144.91`
- 端到端單請求平均速度（估算）
  - `128 / (TTFT + 127 * TPOT)`
  - `128 / (95.0036 + 127 * 0.01259) = 1.33 tok/s`

## 5. 與目標門檻比對

1. 解碼吞吐量（目標 >= 3000 tok/s）
   - 實測：309.15 tok/s
   - 結論：未達標

2. 單請求平均速度（目標 >= 10 tok/s）
   - 實測：約 1.33 tok/s（端到端估算）
   - 結論：未達標

3. 系統處理能力 RPM（目標 >= 1400）
   - 實測：144.91 RPM
   - 結論：未達標

## 6. 結論

本次 0302 多卡（TP=2）重測結果為有效測試（非 0 output token 異常），但三項目標門檻均未達成。
